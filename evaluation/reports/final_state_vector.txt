==============================================================
OVERVIEW
==============================================================

This document describes the complete, structured state vector
that is passed to the RL (Reinforcement Learning) agent in NexResolve.

The state vector is a flat numeric array of 35 features per ticket.
Each feature is a float32 value.

"This structured state vector is the input to the RL agent."

The vector is produced by: rl/state_builder.py
The output dataset is:     data/final/rl_ready_dataset.csv

==============================================================
HOW THE RL AGENT USES THIS VECTOR
==============================================================

The RL agent maps each state vector to one of 4 actions:

  Action 0 — ROUTE      : Direct ticket to appropriate team/queue
  Action 1 — CLARIFY    : Request more information from user
  Action 2 — SUGGEST    : Surface a known solution from KB
  Action 3 — ESCALATE   : Move to higher-tier support

Decision examples using combined NLP + RAG signals:

  HIGH max_sim + LOW gap_flag + HIGH confidence_score
  → Agent has strong KB evidence + clear intent → SUGGEST solution

  LOW max_sim + HIGH gap_flag + HIGH needs_clarification
  → No KB match + missing info → CLARIFY first

  HIGH frustration_level + sla_breach_flag=1 + HIGH gap_flag
  → User angry + SLA breached + no KB match → ESCALATE

  LOW uncertainty_flag + known intent + LOW missing_count
  → Clear intent + complete info + no gap → ROUTE directly
